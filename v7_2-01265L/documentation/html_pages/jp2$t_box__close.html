jp2_input_box::close

virtual bool close( )

Java: public native boolean Close( )

Overrides kdu_compressed_source::close

[Declared in "../apps/compressed_io/jp2.h"]

Go to class description.

Synopsis

Implements kdu_compressed_source::close.

If the box is not currently open, this function does nothing. Otherwise, you need to take note of the following:

  • Although it is OK for a super-box to be closed, if the current object has a super-box, that box's contents are generally accessed by this function, so you should make sure not to destroy the super-box used to open a sub-box, until after the sub-box has at least been closed.
  • The function may attempt to access the underlying jp2_family_src object if it is fuelled by a dynamic cache (i.e., a kdu_cache object), so that the return value reflects the completeness of the reading process as accurately as possible. This could be a problem if there is a chance that the original jp2_family_src object has ceased to exist. To avoid this, you may invoke the close_without_checking function first or instead.

Return Value

True if the contents of the box were completely consumed by calls to read, or by opening and closing of sub-boxes.
