Synopsis

To be completely sure that changes in cache contents are reflected to objects and functions that ultimately source their data from this object, you should call synch_with_cache after the cache contents change. The function does nothing if the object was not opened to read from a kdu_cache object.

The way you would use this function in practice is as follows. When cache contents change significantly, or at episodes that your application determines, if the cache contents have changed, you should first invoke this function and then attempt again to open and read jp2_input_box or derived objects that are of interest, but could not previously be opened or read to completion.
