kdu_params::new_instance

Java: Kdu_params.New_instance

kdu_params * new_instance( )

Java: public native Kdu_params New_instance( )

[Declared in "../coresys/common/kdu_params.h"]

Go to class description.

Synopsis

Adds a new instance to the end of the current object's instance list. The current object need not be the head or the tail of its instance list.

If the parameter class supports tile diversity, new instances must not be added to objects with a negative tile index, except in the event that the object was constructed with treat_instances_like_components equal to true. Similarly, if the parameter class supports component diversity, new instances must not be added to objects whith a negative component index. Finally, new instances may not be added at all if the parameter class does not support instance diversity. See the kdu_params constructor for more on diversity flags.

The present function is primarily intended for adding instances of a parameter class which correspond to successive tile-parts of a particular tile. Instance indices are simply incremented as new instances are appended to the list. If the object was constructed with treat_instances_like_components equal to true, however, you may well wish to add instances in non-linear fashion. For this purpose, you should use the access_relation function.

Return Value

A pointer to the newly created object, or NULL if a new instance can or should not be created for any reason.
