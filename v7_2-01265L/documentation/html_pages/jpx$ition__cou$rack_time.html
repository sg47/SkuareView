Synopsis

Similar to count_track_frames, except that this function assesses the duration of the indicated track, measured in multiples of 1/TS seconds, where TS is the value returned by get_timescale — in practice the measurements are all currently in milliseconds.

If the actual track duration might be larger than the value reported here, the function returns false — a larger value might be returned if the function is called again later, after more data is accumulated in a dynamic cache.

This function is intended for use with composition objects obtained via jpx_source::access_composition. In all other circumstances, the function returns false, setting count to 0.
