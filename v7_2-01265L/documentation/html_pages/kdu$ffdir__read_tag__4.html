Synopsis

Same as the second form of read_tag, except that this function may be used only with tags whose data type (least significant two bytes of the tag_type) is one of 5, 10, 11 or 12. These are the data types for which each entry represents a floating point quantity, either as a signed/unsigned rational, or as an IEEE single/double precision float. Attempting to read anything else will generate an error through kdu_error.

Note that this function correctly handles byte order and precision conversions.

Return Value

Actual number of values read.

Arguments

tag_type [kdu_uint32]

length [kdu_long]

Number of doubles to read.

data[] [double]

Array containing at least length entries, into which the data is written.
