[Declared in "../apps/support/kdu_region_compositor.h"]

Go to class description.

Synopsis

Removes all elements from the jitter-absorbtion queue managed by push_composition_buffer and pop_composition_buffer.

As discussed in push_composition_buffer, for design patterns involving a processing master thread and a display master thread, the processing thread should be the one that invokes this function, but it should do so only when it can be sure that the display thread is inactive or appropriately locked, so that flushed buffers are not being used for display purposes.
