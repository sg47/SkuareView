Synopsis

Same as the first form of write_tag, except that this function may only be used with tags whose data type (least significant two bytes of the tag_type) is one of 3 or 8. These are the data types for which each entry is 2 bytes long. Attempts to write anything else will generate an error through kdu_error.

Note that this function adjusts the byte order of each field written in accordance with the native machine byte order, so that the returned 16-bit integers will be correctly interpreted.

To pass signed 2's complement quantities (data type 8) using this function, it is sufficient to cast the data array to one of type kdu_int16.

Arguments

tag_type [kdu_uint32]

length [int]

Number of 16-bit words to write.

data[] [const kdu_uint16]

Array containing at least length entries, from which the data is written.
