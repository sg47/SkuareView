Synopsis

Implements kdu_compressed_target::start_rewrite. Most jp2_output_box objects can support rewriting. This is certainly true if the box contents are being buffered in memory. It is also true for top-level boxes whose jp2_family_tgt object offers repositioning services (either because it was opened with a file, or because it was opened with another jp2_compressed_target object which itself supports rewriting). However, there can be circumstances where JP2 boxes will not support rewriting. For example, if a box's ultimate target does not support repositioning and the box's length has already been signalled using set_target_size, calls to the present function will have to return false.

If a box has been closed, you cannot use this function; however, you can use the reopen function, which internally uses the same services as start_rewrite and end_rewrite. However, start_rewrite will not succeed if you attempt to invoke it directly on a box that has been re-opened using reopen.
