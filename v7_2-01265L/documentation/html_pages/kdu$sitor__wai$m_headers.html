[Declared in "../apps/support/kdu_region_compositor.h"]

Go to class description.

Synopsis

This function is provided because set_frame and add_ilayer can succeed, even if the relevant codestream headers are not available at the time when they are called. This allows buffers to be allocated, set_scale, get_total_composition_dims, process and many other functions to be used successfully, even if no codestream content or even codestream headers are available yet. However, the results produced by some functions are not necessarily reliable until the codestream headers are available. The most important example of this is the find_compatible_jpip_window function, which may under-estimate or over-estimate the size of the JPIP request window that should be issued for a given scale (as set using set_scale) until all relevant codestream main headers have become available.

If the present function returns false, you can be sure that all codestream headers are loaded and sufficient information is available to reliably map regions to/from the codestream, access the codestreams, construct the most compatible JPIP window request, and so forth.
