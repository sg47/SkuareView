Synopsis

If the object was opened using a dynamic cache, this function provides the most computationally efficient means for determining whether or not the main header of the indicated codestream is complete; this can save the cost of opening a jp2_input_box and testing for completeness. The function does not need to acquire any mutual exclusion locks and its return value is always valid, no matter what else is going on.

If the object was not opened using a dynamic cache, this function always returns true, without checking for existence of the indicated codestream within the file.

logical_codestream_idx [kdu_long]
