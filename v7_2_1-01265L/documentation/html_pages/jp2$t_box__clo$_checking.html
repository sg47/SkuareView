jp2_input_box::close_without_checking

Java: Jp2_input_box.Close_without_checking

virtual void close_without_checking( )

Java: public native void Close_without_checking( )

[Declared in "../apps/compressed_io/jp2.h"]

Go to class description.

Synopsis

This function may be invoked instead of close if you are not interested in the return value from close. The function does itself invoke the virtual close function, so you can be sure that derived classes that override close will have that function invoked by this one. However, the function first sets up the internal state of the object in such a way as to be sure that the underlying jp2_family_src object is not accessed during the call to close — otherwise, a direct call to close may access this function to check whether the contents of the box were complete in a dynamic cache ( kdu_cache object), so that it can provide a reliable return value.

Call this function is you cannot be certain that the jp2_family_src object still exists at the time when the box needs to be closed.

Note that this function is called implicitly from the jp2_input_box object's destructor.
