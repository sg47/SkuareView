Synopsis

Same as the first form of the overloaded init function, except that the colour space is defined by an embedded ICC profile, supplied via the buffer pointed to by the icc_profile argument. The internal object makes a copy of this buffer.

The standard ICC header in this buffer identifies its length and other attributes, from which the object determines whether it conforms to the restricted ICC profile or not. It is best, if possible, to provide at least one colour space description which conforms to the simple ICC profile, or one of the enumerated colour spaces, JP2_sRGB_SPACE, JP2_sYCC_SPACE or JP2_sLUM_SPACE, since these are JP2-compatible.

To conform to the restricted ICC profile compatible with JP2, the profile must conform to one of the following classes of ICC profiles:

  • The monochrome input class;
  • The monochrome display class;
  • The matrixed 3-colour input class; or
  • The matrixed 3-colour display class.

These profiles must all be defined with respect to the XYZ Profile Connection Space (PCS_XYZ).

Note that input and display classes are often functionally equivalent. Note also that the admissibility of display classes is formally introduced by AMD 6 to IS15444-1, prior to which only input classes were admissible. Older implementations of JP2 file format reading/writing software might reject profiles that use the display class.
