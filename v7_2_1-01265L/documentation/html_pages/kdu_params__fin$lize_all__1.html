Synopsis

This function invokes the finalize member function of the current object. If the object is an instance head, all objects in the instance list are finalized. If the object has a component index of -1, all objects with the same tile index are also finalized. If it has a tile index of -1, all objects with the same component index are finalized. If it is a cluster head, all objects in the same cluster are finalized, and if it is the head of the list of clusters, all objects in all clusters are finalized.

This function (and especially the second version below) also plays an important role in dealing with parameters that have been parsed from one codestream and are subsequently reparsed from a different codestream, with potentially different parameters. This is relevant only if after_reading is true, in which case the function examines the kd_attribute::num_marked_records values to determine whether any attributes have more records than the number that are marked. This means that more records were set for the parameter prior to a call to clear_marks than were set after the call (this happens for codestreams that have been restarted, using kdu_codestream::restart). When this condition is detected, the kd_attribute::num_records value is reduced to kd_attribute::num_marked_records, erasing any surplus attribute values that were read from a different codestream, prior to the clear_marks call. At the same time, the changed flags are manipulated so as to ensure that the parameter change becomes apparent to the kdu_codestream machinery.

Arguments

after_reading [bool]

If true, the finalization process is being performed on parameters which were obtained by reading codestream marker segments. This can affect the behaviour of individual derived finalize functions, and also allows them to minimize their effort, since complex finalization is often not required in this case. The function is invoked with this argument equal to true from within the codestream management system — you would not normally invoke it in this way directly from within an application.
