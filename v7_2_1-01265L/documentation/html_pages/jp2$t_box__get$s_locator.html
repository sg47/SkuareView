jp2_input_box::get_contents_locator

Synopsis

Returns a locator for the first byte of this box's contents. The return value may be invalid if the box is not open. The file position (as returned by jp2_locator::get_file_pos) of the returned locator should differ from that of the locator returned by get_locator by exactly the header length, as returned by get_box_header_length. However, the databin-id of the returned locator (see jp2_locator::get_databin_id) may be very different in the event that the ultimate source of data is a dynamic cache and the box is recovered from a placeholder, which identifies a different data-bin for the box contents.

Arguments

class_id [kdu_int32 *]

If non-NULL, this is used to return the class of any data-bin associated with the returned locator. The data-bin's id and class may be used together with the relevant location within the data-bin in a call to kdu_cache::set_read_scope. The main application for this function, however, is in the construction of JPIP client metadata requests.
