Synopsis

Same as the second form of write_tag, except that this function may be used only with tags whose data type (least significant two bytes of the tag_type) is one of 5, 10, 11 or 12. These are the data types for which each entry represents a floating point quantity, either as a signed/unsigned rational, or as an IEEE single/double precision float.

Note that this function correctly handles byte order and precision conversions.

Arguments

tag_type [kdu_uint32]

length [int]

Number of double to write.

data[] [const double]

Array containing at least length entries, from which the data is written.
