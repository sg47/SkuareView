Synopsis

If you use this constructor, you must issue a subsequent call to create before using other member functions. The env and env_queue arguments are used to initialize the multi-threading state which may subsequently be changed using the set_thread_env function — see that function for an explanation.

If you are supplying a non-NULL env argument to this constructor, we strongly recommend that you read the description of the set_thread_env function, since all the constraints that are described there also apply when the object is constructed directly with a non-NULL kdu_thread_env reference. In particular, it is IMPERATIVE that all access to this object's interface functions be restricted to the thread that is associated with the non-NULL env object, at least until set_thread_env is called again with a different (or NULL) env argument.

If you never retract the multi-threading environment supplied via the env argument (can be done by passing NULL to set_thread_env), you should be careful to destroy the present object before destroying the env object, unless you take steps to first close all internal open codestreams by using remove_ilayer with ( permanent=true).
