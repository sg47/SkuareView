Synopsis

If the compressed video source provides no timing information, this function returns 0. Otherwise, it returns the number of ticks associated with the frame to which the currently open image belongs. If no image is currently open, the function returns the frame period associated with the frame to which the next open image would belong if open_image were called. The number of ticks per second is identified by the get_timescale function. If the video is interlaced, there are two images (fields) in each frame period.
