Synopsis

This function is primarily intended for writing boxes that have been opened without any super-box or jp2_family_tgt of their own directly into a specified jp2_family_tgt object.

If force_headerless is true, the behaviour of the function is at least conceptually the same as using get_contents to access the contents buffer and then writing these contents directly to tgt; however, jp2_family_tgt deliberately provides no writing interface of its own. If force_headerless is false and the box was not opened as headerless, the box relevant header is also written to tgt regardless of whether the current object was opened as headerless or not.

In the future we will support jp2_family_tgt objects that are opened for writing directly to a kdu_cache; in this case, box writing will involve the creation and writing to cache data-bins and box header writing will involve the writing of placeholder data-bins. In such circumstances, it might not be possible to access the contents of the box as an opaque continuous span of data, using get_contents; however, it will be possible to use this function.

Return Value

Total number of bytes written, including the box header if it is written. If the function failed to write all required bytes to the tgt object, it returns -1.
