jp2_box_textualizer::textualize_box

Synopsis

This function locates the relevant textualizer function for the supplied box and writes a textual description of the box to the supplied tgt object. The four arguments to this function are identical to those passed to the textualizer function that was originally supplied to add_box_type.

Return Value

False if no text was generated, for one of the following reasons:

  • No call to add_box_type supplied a box type that matches the value returned by box→get_box_type; or
  • The matching call to add_box_type had a NULL textualizer_func argument; or
  • The relevant textualizer function itself returned false.

In the event of a false return, the caller might choose to print a hex dump of the box's contents.

Arguments

box [jp2_input_box *]

The read pointer of the box is expected to be positioned at the start of its contents on entry to this function. On exit, the box→get_remaining_bytes function can be used to determine how many of the box's bytes were actually consumed by the textualizer function — this information can be used to determine whether or not it is appropriate to print an additional line of text such as <unexpanded bytes="21" />.

The textualizer functions should generally be able to cope with boxes that are not yet complete — i.e., their box→is_complete function may return false. In that event, the function should expand as much of the available contents into text as possible.

tgt [kdu_message &]

Object to which the text description is directed.

xml_embedded [bool]

This value is passed as the third argument to the box textualizer function. If true, the textualizer function is notified that the text it generates will form the body of an XML elements, so some effort should be employed to make the text XML-legal. In most cases, textualizers should aim to express the contents of the box as XML elements, so there should be no problem. However if the box contains plain text or an XML document, the textualized version produced by this function should generally be embedded in an XML CDATA section. If this argument is false, there is no need for the CDATA section.

max_len [int]

This value is passed as the fourth argument to the box textualizer function. Generally box textualizer functions will either ignore this argument and textualize the entire box contents, or else they will interpret this argument as a limit on the number of bytes from the box that should be textualized.
