Synopsis

Implements the policy described in check_continue. Specifically, this function accumulates positive values of new_dependencies in an internal state variable, assuming that all calls that involve such positive values arrive from within the thread that is currently executing the "processor" job function described in the introductory comments to kdu_run_queue. You do need to be extremely careful that once check_continue returns false, your "processor" function returns immediately, without manipulating the state of this or any other object that is not inherently thread safe, since a new copy of the processor function could be running in another thread from that point onwards.

For efficiency reasons, this function ignores delta_max_dependencies and does not pass dependency information along to any parent queue. For this reason, instance of this object will usually be attached as top-level queues within the relevant thread group.
