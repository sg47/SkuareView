kdu_overlay_params::get_max_painting_border

Synopsis

Returns a copy of the max_painting_border parameter supplied in the most recent call to kdu_region_compositor::configure_overlays. The interpretation of this parameter is the maximum amount by which the painted overlay content may exceed the boundaries of the mapped ROI node, in each direction, as measured on the compositing reference grid.

Overlay data for any given node' may optionally be painted with a smaller border, if desired. The border is used internally to size the bounding_region passed to kdu_region_compositor::paint_overlay and kdu_region_compositor::custom_paint_overlay, and also to determine which overlay segments intersect with a node's region on an overlay buffer, so you should not paint with a larger border.
