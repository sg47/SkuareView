Go to class description.

Synopsis

See 16-bit version for an explanation. Note that the raw_buf array is expected to be aligned on a 32-byte boundary and have storage that spans raw_width samples, rounded up to a whole number of octets. That buffer should have been allocated using the same kdu_sample_allocator as the current object.
