jp2_input_box::fork

Java: Jp2_input_box.Fork

Java: public native void Fork( Jp2_input_box src)

[Declared in "../apps/compressed_io/jp2.h"]

Go to class description.

Synopsis

This function is similar to transplant, in that the present object is opened (most not be open already) so as to present the same state and read position as the src object, except that the src object is not closed.

As with transplant, the only difference between the present object's new state (on exit) and the src object's state is that the present object does not retain any connection with a containing super-box. The src object retains any such connection, meaning that its super-box (if any) cannot be used to directly open any further sub-boxes or read content data until the src object has been closed.

Note: if src.load_in_memory has been invoked, the function will leave two copies of the original box's contents in memory. This saves the cost of reading the box's contents again from its source, but could become expensive if the box is very large. On the other hand, if src is a sub-box of a box that has been loaded into memory, the present box will not automatically be loaded into memory and will have to fetch its data from the original data source.
