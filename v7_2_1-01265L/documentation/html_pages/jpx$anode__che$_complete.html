Synopsis

This function examines the immediate descendants (i.e., children) of the current node, returning true if the number of children is known, every child's box-type at least is known, get_descendant would return a non-empty interface for every descendant whose box type is found in the box_types_of_interest array, and get_link would at least identify the correct link-type for every descendant whose box type is found in the box_types_of_interest array.

This function may be extended in the future with recursive capabilities, but it is not currently clear how such recursion should be be defined with respect to boxes of interest. For this reason, we defer such an extension until its application becomes clear.

Arguments

num_box_types_of_interest [int]

Number of elements in the box_types_of_interest array. If zero, the caller is interested in all box-types.

box_types_of_interest[] [const kdu_uint32]

Array of box types which are of interest to the caller; may be NULL if num_box_types_of_interest is 0.
