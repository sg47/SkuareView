jp2_output_box::use_long_header

[Declared in "../apps/compressed_io/jp2.h"]

Synopsis

Forces the box header to be written with an 8-byte length field, even if 4 bytes would have been enough. This function can be called any time prior to the actual writing of the box header. The function conveniently returns the actual length of the box header, which will be 0 if the box was opened as header-less (see open). The main reason for providing this function is to allow get_header_length to return a known value before the contents of the box can be known. Of course, this function does nothing if the box was opened as header-less — see open.
