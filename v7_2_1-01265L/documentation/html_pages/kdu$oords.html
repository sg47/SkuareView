Synopsis

Manages a single pair of integer-valued coordinates. To manage a pair of coordinates, you should generally use kdu_coords, rather than this object. However, kdu_coords involves a constructor, so you may not be able to use it in all circumstances (e.g., when embedding a coordinate pair within a C++ union). This object has no default constructor at all, so is easy to embed into other objects without requiring invocation of a constructor. The kdu_coords object is also equipped with copy constructor and assignment operators, which can accept kdu_nc_coords as input, so you should be able to pass a kdu_nc_coords instance to any function which normally requires a kdu_coords object.
