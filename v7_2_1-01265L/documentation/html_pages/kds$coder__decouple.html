kds_id_encoder::decouple

virtual void decouple( )

[Declared in "../apps/kdu_server/kdu_serve.h"]

Go to class description.

Synopsis

After this function is called, the next data-bin ID to be encoded by the encode_id function will be entirely independent of all previous headers. Otherwise, the header may be coded differentially with respect to previous headers. This function is invoked automatically, at the start of each call to kdu_serve::generate_increments, which ensures at least that the data generated from that call can be decoded unambiguously without reference to the information produced by other calls to kdu_serve::generate_increments, possibly with a different window context (the kdu_serve object can manage multiple concurrent window contexts). The decouple function may be called more frequently, as explained in connection with the decouple_chunks argument to kdu_serve::generate_increments.
