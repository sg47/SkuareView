Synopsis

Causes region to be added to the internal machinery which keeps track of rectangular regions which need to be included in regions returned by process, before is_processing_complete can return true. The explicit invalidation of regions at most causes them to be recomposited onto a compositing surface — this is quite different to refresh, which results in complete re-rendering of the content.

The function is provided for the benefit of applications which need to repaint some portion of a composition to their display buffer, but want the repainting to be driven by returns from the process function so that a region whose underlying imagery has yet to be decompressed and rendered will not be repainted prematurely.

Arguments

region [kdu_dims]
