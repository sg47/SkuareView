Synopsis

The add_thread function uses this function to create new kdu_thread_entity objects for each new worker thread. You can override this virtual function to create objects of your own derived class, thereby ensuring that all worker threads will also use the derived class. This is particularly convenient if you want each thread in a group to manage additional thread-specific data. By careful implementation of the function override, you can also arrange to inherit parameters from the thread which creates new workers.
