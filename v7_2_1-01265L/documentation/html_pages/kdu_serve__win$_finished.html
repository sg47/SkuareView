Synopsis

This function may be called to clean up temporary resources that may have been set up during calls to generate_increments. This can be particularly beneficial when there are multiple window contexts in use. For that reason, the context_id argument to this function deliberately has no default.

Typically, you would call set_window, followed by one or more calls to generate_increments. If a new window of interest becomes available (typically from a client request), you may just install the new window with a call to set_window and then continue calling generate_increments. On the other hand, if there is no new window of interest and all relevant data has been generated by calls to generate_increments (i.e., if get_window returns false), it may be advisable to call this function to clean up resources. Note carefully that even if get_window returns false, it is possible that a future call to release_chunks returns some abandoned data chunks so that future calls to generate_increments with the same window of interest will return more data. This situation can be desirable in the context of lossy transmission schemes.

context_id [int]
