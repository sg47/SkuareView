Synopsis

If you wish to learn the number of bytes written to each packet within a precinct, you should invoke this function from within the post_end_precinct call — at any other point, the function will just return 0. Within post_end_precinct, the function returns the minimum of num_packets and the number of packets that was passed to the post_end_precinct function, and writes this number of length values to the packet_lengths array, starting with the length of the first packet in the precinct.

Arguments

num_packets [int]

packet_lengths[] [kdu_long]
