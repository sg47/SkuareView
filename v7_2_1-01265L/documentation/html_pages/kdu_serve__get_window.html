kdu_serve::get_window

Java: Kdu_serve.Get_window

bool get_window( kdu_window & window, int context_id=0)

Java: public native boolean Get_window( Kdu_window window, int context_id)

[Declared in "../apps/kdu_server/kdu_serve.h"]

Go to class description.

Synopsis

Retrieves the window parameters which are to be used in the next call to generate_increments unless set_window is called first. As with most functions offered by kdu_serve, this call relates to a specific window context, as identified by context_id.

The principle reason for calling this function is to learn about any modifications which the server made to the window parameters supplied in a previous call to set_window. As noted in connection with that function, the server may need to make such changes in order to satisfy its own resource constraints. In particular, very large windows (large spatial regions and/or large number of components) may require excessive memory to be allocated by the server.

It is important to realize that the server may not make any necessary alterations to the window parameters supplied to set_window until after the first subsequent call to generate_increments. Thus, the most meaningful point at which to call the present function is after the first (or a subsequent) call to generate_increments which follows a set_window call.

Return Value

True if the window is still incomplete, in the sense that not all data which is relevant to the window has yet been packaged up in calls to generate_increments. Note, however, that even if this function returns false, meaning that the window is complete, future calls to release_chunks may provide information about generated data chunks which have been abandoned, as a result of which a call to the present function may return true and future calls to generate_increments may be required to regenerate some of the abandoned data.

Arguments

window [kdu_window &]

context_id [int]

As explained in connection with set_window, this object is capable of managing multiple concurrent window contexts. If no context with the indicated context_id has yet been used, the present function invokes window.init and returns false without creating a new window context.
