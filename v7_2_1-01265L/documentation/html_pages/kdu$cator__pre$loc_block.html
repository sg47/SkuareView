kdu_sample_allocator::pre_alloc_block

size_t pre_alloc_block( size_t num_bytes)

[Declared in "../coresys/common/kdu_sample_processing.h"]

Go to class description.

Synopsis

This function is provided to allow for allocation of general storage, a little bit like malloc. The pre-allocated block of memory is aligned at least on an 8-byte boundary, although the default alignment might be stricter as a matter of system-wide optimization. In any case, you can enforce stricter alignment by calling pre_align prior to this function.

Remember that you can legally read from (but not write to) locations that lie up to 128 bytes before and 128 bytes after the nominal allocated memory block.

Return Value

The returned value is an allocation offset that must be remembered by the caller and passed to the alloc_block function as appropriate. This allows calls to alloc16, alloc32 and alloc_block to safely proceed in a multi-threaded environment without the need for thread synchronization.

Arguments

num_bytes [size_t]
