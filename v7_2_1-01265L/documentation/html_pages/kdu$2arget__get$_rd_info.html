Synopsis

The serve target might recover additional information concerning rate-distortion properties of a codestream and make this information available through the present function. This information is supposedly recovered without physically opening the codestream every time. This means that the server can use this information to determine how to distribute an available bit budget over a potentially large number of codestreams and then open them sequentially to actually serve the data.

If layer_log_slopes is non-NULL, the value of num_layer_slopes on entry is taken to hold the size of the layer_log_slopes array, so that no more entries will be written than this. Similarly, if layer_lengths is non-NULL, num_layer_lengths on entry holds the size of the array. In any case, upon return the values of num_layer_slopes and num_layer_lengths hold the number of quality layers for which rate-distortion slope thresholds are available and the number of quality layers for which compressed lengths are available, respectively. Either or both of these values may be 0.

The quantities returned via the layer_log_slopes array represent 256*(256-64+log_2(S_L)) where S_L is the distortion-length slope threshold for quality layer L, measured as the ratio between the reduction in total squared error distortion (with samples normalized to the range 0 to 1.0) and the increase in code length (measured in bytes). These values should generally lie in the interval 0 to 65535, even if they are returned as integers. However, this range might occasionally be violated in order for the internal machinery to guarantee that the log slope values decrease montonically — that may require some adjustment to the originally recorded values.

Return Value

False if the codestream does not exist or if no R-D information is available for it.
