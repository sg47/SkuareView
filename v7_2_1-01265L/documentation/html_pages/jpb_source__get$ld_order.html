[Declared in "../apps/compressed_io/jpb.h"]

Go to class description.

Synopsis

See kdu_compressed_source::get_field_order. Note that the elementary broadcast stream specification allows the field order (and the number of fields) to change from frame to frame, although this seems highly unlikely in practice. Also, it can happen that the value returned by this function is KDU_FIELDS_UNKNOWN, meaning that the content is interlaced, but the order of the fields is not actually known. Hopefully, this should not usually happen in practice.
