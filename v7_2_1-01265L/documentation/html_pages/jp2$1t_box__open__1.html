Synopsis

Opens a new output box with the indicated box_type (usually a 4 character code). If the rubber_length and headerless flags are both false, the contents of the box will be buffered so that the final length of the box may be determined and written as the first field of the box's header. Otherwise, the box will pass its contents directly through to the tgt object as they appear. Note that a box may be given a rubber length after it is opened, using set_rubber_length. The write_header_last function may also be used to allow boxes with headers to be written on the fly. None of these have any effect if headerless is true, in which case the box has a nominal box type and a length, but no header is written to identify either the type code or the length. Headerless boxes are typically not used at the application level, but may be used internally within a file format implementation, in order to realize certain advanced file structures.
