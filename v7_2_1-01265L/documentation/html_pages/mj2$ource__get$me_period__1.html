Synopsis

Returns the number of ticks associated with the frame to which the currently open image belongs, or would belong if open_image has still to be called. If no image is open and the the call to open_image would fail, the function returns 0. If this happens, you can rectify the problem by seeking to a valid frame before calling the function again. The number of ticks per second is identified by the get_timescale function. If the video is interlaced, there are two images (fields) in each frame period.
