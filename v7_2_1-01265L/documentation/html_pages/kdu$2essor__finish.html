kdu_stripe_decompressor::finish

Java: Kdu_stripe_decompressor.Finish

bool finish( )

Java: public native boolean Finish( )

[Declared in "../apps/support/kdu_stripe_decompressor.h"]

Go to class description.

Synopsis

Each call to start must be bracketed by a call to finish, if you wish to re-use the object to decompress further code-streams. Calling this function is not strictly necessary if you intend to use the function only once.

In the case of multi-threaded processing (non-NULL env argument passed to start), after this function returns there is guaranteed to be no ongoing processing associated with the tile processing engines that were created by this object. However, it is possible that the codestream itself still has ongoing background processing. For this reason, you should remember to call kdu_thread_env::cs_terminate prior to kdu_codestream::destroy, regardless of whether the present function returns true or false.

Return Value

True only if all available image data was recovered using the pull_stripe function.
