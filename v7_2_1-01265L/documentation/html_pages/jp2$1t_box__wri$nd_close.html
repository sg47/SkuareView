Synopsis

This is a convenience function that writes a sub-box of type jp2_free_4cc at the current location, having a total box length (including the header) of free_bytes bytes and then closes the current box. The contents of the free box are undefined, but that should not matter to any caller of this function.

One convenient feature of the function is that whether or not a long header (16 bytes) needs to be written is taken into account in determining the number of body bytes to write to the terminal free sub-box.

A second convenient feature of the function is that the current box's header and existing contents can be written immediately, after which the free sub-box can be written. If free_bytes is very large, this can save considerable buffering of meaningless box contents in memory.

A third feature is that if the box (or a super-box) is currently being re-written or has been re-opened, the current function writes only the free box header and need not write anything to the body of the final free sub-box — it needs only to skip over the body locations that already have some previously written content.

free_bytes [kdu_long]
