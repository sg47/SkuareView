Synopsis

Use this function to probe the descendants of a node. If this is a leaf node, there will be no descendants so all calls to this function will return empty interfaces. Otherwise, there will be anywhere from 1 to 4 children.

If the node is split only in the horizontal direction, non-empty interfaces will be returned when child_idx is one of LL_BAND or HL_BAND.

If the node is split only in the vertical direction, non-empty interfaces will be returned when child_idx is one of LL_BAND or LH_BAND.

If the node is split in both directions, non-empty interfaces will be returned when child_idx is one of LL_BAND, LH_BAND, HL_BAND or HH_BAND.

In the degenerate case of no splitting at all, only the LL_BAND child exists. This can happen for a primary node in a degenerate resolution level, whose next resolution has exactly the same dimensions — JPEG2000 Part-2 DFS marker segments can cause this to occur if a splitting code happens to be 0.

The LL_BAND child of a primary node is always the primary node of the next lower resolution object, unless that resolution would be the lowest resolution of all (the one with res_level=0), in which case the LL_BAND child is a subband — i.e., its access_subband function will return a non-empty interface. This means that you can use the access_child and access_resolution functions as an alternate mechanism for navigating to each of the available resolutions in a tile-component, except for the lowest resolution.

Arguments

child_idx [int]
