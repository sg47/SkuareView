kdu_region_compositor::load_metadata_matches

Go to class description.

Synopsis

This function does nothing unless the object was initialized with a jpx_source object. In that case, the function invokes jpx_meta_manager::load_matches passing in arrays that identify all codestreams and compositing layers that are in active use. If you are interested in all possible metadata, for some reason, it is better to directly call jpx_meta_manager::load_matches with -1 for the arguments that identify the number of codestreams and compositing layers of interest, since that will expand all codestream and compositing layer contexts for newly arrived metadata. However, for a source with a massive number of codestreams or compositing layers, that could be very tedious.

Return Value

True if and only if the call to jpx_meta_manager::load_matches returns true.
